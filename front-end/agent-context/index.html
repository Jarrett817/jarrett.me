01 · 当前对话

大文件读取：分块续读

pi 的 read 从磁盘读全文件后切片；单次 toolResult 硬上限 2000 行或 50KB，超出部分附带 Use offset=… to continue 提示，由模型多轮续读。

续读时模型通常只传 offset，不传 limit——由 pi truncateHead 自动截到 2000 行；limit 仅在精读等场景由模型自选。

磁盘
app.ts
共 8400 行（全量在磁盘）
模型发起
read("app.ts")
进上下文 toolResult
第 1–2000 行原文
[Showing lines 1-2000 of 8400. Use offset=2001 to continue.]

01 · 当前对话

大文件读取：关键词定位

pi grep 调 ripgrep，返回 文件:行号: 内容（默认 100 条匹配、单行 500 字符）； 需要更大范围时，模型可再 read(offset, limit)——这是常见策略，不是 grep 内置第二步。

pi grep 底层调 ripgrep，默认最多 100 条匹配、单行 500 字符、总输出 50KB；结果已含 文件:行号: 内容，不是只返回行号。

01 · 当前对话

大文件读取：尾部场景

bash 输出超限：pi 统计 stdout+stderr，超 2000 行或 50KB 自动 truncateTail；读日志末尾由模型发 tail；不够则 read 临时文件或扩大范围。

模型只发 bash 命令

模型调用 bash 工具执行命令。pi 在后台用 OutputAccumulator 流式收集 stdout + stderr 文本，模型不参与长度判断。

01 · 当前对话

截断后怎么衔接

三件事：工具调用和返回要成对；裁切按行不按词；裁掉的部分靠续读提示和压缩摘要补——pi 保证格式不乱，不保证模型一次看懂全部。

这里说的是：发给 LLM 的对话记录格式不能坏。pi 会保证「调了工具」后面一定有「工具返回」，压缩时也不会从返回中间切开。 ——不等于模型看懂了全部内容。

调了工具，就必须有返回
模型说「我要 read 某文件」→ 下一条消息一定是这次 read 的返回。不能只有调用、没有结果。
压缩对话时，不从返回中间切
context 满了要压缩旧消息时，切点只能落在用户话或模型话上，不会把某次工具返回切成上下两半。
一次工具返回是一条完整消息
read/bash/grep 就算内容被截短，也仍算同一条返回；不会拆成两条 tool 消息。

context window 内的信息管理：主动压缩、策略组合与 keepRecent

02 · 窗口压缩

主动压缩

工具返回过长时，在写入 messages 前先做摘要压缩（例如 2000 token → 约 200 token），避免被动触发窗口压缩。

02 · 窗口压缩

记忆压缩方法

常见组合：滑动窗口 + 摘要压缩。亦可叠加重要性过滤与结构化抽取。

仅保留最近 N 轮；实现简单，按时间截断，不区分内容重要性。

编码 Agent 常用作兜底：达到窗口硬上限时触发。

Prompt Caching（计算层）与记忆压缩（信息层）互补，不互相替代。

02 · 窗口压缩

实现：keepRecent 切分

切点之后保留原文；更早历史合并为摘要；右侧预留回复 token。

摘要 ~92k
原文 ~20k
预留 16k

窗口 128k · 触发：已用 tokens > 128k − 16k = 112k

触发条件：已用 token > 窗口上限 − 回复预留

03

跨轮记忆

窗口外持久化：写入标准、类型划分、粒度与读回路径

03 · 跨轮记忆

写入标准

仅持久化对后续任务有增量价值的信息。

建议写入
  • 用户偏好与习惯
  • 关键结论与决策
  • 项目规则 / 外部知识
不建议写入
  • 中间推理过程
  • 原始 tool 大输出
  • 一次性噪声与闲聊

03 · 跨轮记忆

跨轮记忆类型

任务结束后 messages 清空；跨轮数据落盘后，在拼上下文阶段注入。

当前分支消息，每轮拼入

03 · 跨轮记忆

存储粒度

粒度过细 — 单条偏好拆成多条存储，检索时易部分命中，信息不完整。
粒度过粗 — 整段任务历史单条存储，检索命中后无关内容占比高。

推荐粒度：一次完整交互 或 一条独立知识点。

03 · 跨轮记忆

持久化与读回

磁盘：1 行 JSON = 1 entry。读回：转为 message 或摘要块后拼入上下文。

记忆生命周期

读（任务开始注入）→ 用（执行中维持 messages）→ 写（任务结束落盘）。

任务开始：按任务描述检索摘要、规则与历史偏好，注入 system prompt。

  • 当前对话：本轮 LLM 可见的全部输入
  • 窗口压缩：context window 内的信息管理策略
  • 跨轮记忆：窗口外持久化，按需注入
