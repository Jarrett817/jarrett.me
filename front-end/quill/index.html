基于 Quill.js 拆解富文本编辑器设计思想 ​

你会怎么设计一个富文本编辑器？ ​

  • 数据驱动
  • 模块化架构可插拔
  • 一致性

思考：如何表示富文本内容？

  • 符合直觉的树形结构
json
// 我们可能想到的结构
{
  "id": "132123123",
  "type": "paragraph",
  "content": "Hello World",
  "format": { "bold": true },
  "children": [
    {
      "id": "132123123",
      "type": "paragraph",
      "content": "Hello World",
      "format": { "bold": true }
    }
  ]
}

实际实现：

json
// Slate.js - 树形嵌套结构
[
  {
    "type": "paragraph",
    "children": [
      { "text": "Hello " },
      { "text": "World", "bold": true }
    ]
  }
]

// TipTap - 基于 ProseMirror 的 JSON 文档
{
  "type": "doc",
  "content": [
    {
      "type": "paragraph",
      "content": [
        { "type": "text", "text": "Hello " },
        { "type": "text", "marks": [{ "type": "bold" }], "text": "World" }
      ]
    }
  ]
}

思考：如何设计架构？

  • 模块化、可插拔
ts
// 可能的设计
class Editor {
  plugins: Plugin[] = [];
  registerPlugin(plugin: Plugin) {
    this.plugins.push(plugin);
  }
}

实际实现：

ts
// TipTap - 扩展系统
import { Editor } from '@tiptap/core';
import Bold from '@tiptap/extension-bold';
import Image from '@tiptap/extension-image';

const editor = new Editor({
  extensions: [Bold, Image.configure({ inline: true })]

// Quill - Blot 模块系统
import Quill from 'quill';

// wangEditor - 菜单和命令分离
editor.config.menus = ['bold', 'italic', 'image'];

为什么模块化？

体积可控、功能灵活、易于扩展

跨浏览器一致性处理 ​

ts
// Quill - Parchment 抽象层统一 DOM 操作
class BoldBlot extends InlineBlot {
  static create() {
    // 统一创建 <strong>，不依赖浏览器默认行为
    return document.createElement('strong');
  }
  // 所有浏览器都使用相同的标签和结构
}

// Slate - 操作抽象 + 标准化选区
Transforms.setNodes(editor, { bold: true });
// 内部统一处理：
// 1. 标准化选区（normalizeSelection）
// 2. 统一 DOM 操作（不依赖 execCommand）
// 3. 处理浏览器差异（Chrome/Firefox/Safari）

// TipTap/ProseMirror - Schema 规范化
const schema = new Schema({
  marks: {
    bold: { parseDOM: [{ tag: 'strong' }, { tag: 'b' }] }
    // 统一将 <b> 和 <strong> 解析为 bold mark
  }
});

基本与我们的设想一致 😀 ​

quill 是什么？ ​

一个高度可定制的免费开源富文本编辑器

启用一个 quill 编辑器 ​

js
const quill = new Quill('#editor', {
  modules: { toolbar: true },
  theme: 'snow'
});

一、概述 ​

核心设计理念 ​

  • 数据驱动：Delta 数据模型
  • 模块化架构：易于扩展和定制
  • 跨浏览器一致性：Parchment 抽象层

二、Delta 数据模型 ​

Delta = 极为紧凑的 JSON 数据结构

json
{
  "ops": [
    { "insert": "Hello " },
    { "insert": "World", "attributes": { "bold": true } },
    { "insert": "\n" }
  ]
}

Delta 描述富文本 ​

json
{
  "ops": [
    { "insert": { "image": "https://quilljs.com/assets/images/logo.svg" } },
    { "insert": "\n" },
    { "insert": { "formula": "e=mc^2" } }
  ]
}

可以自定义任何富文本内容和格式，比如思维导图、3D 模型等等

Delta 的三种操作类型 ​

操作说明示例
insert插入内容{ "insert": "text" }
retain保留并格式化{ "retain": 5, "attributes": { "bold": true } }
delete删除内容{ "delete": 3 }

监听text-change事件，可以看到 quill 对用户每一步操作的定义

Delta 格式与操作变换的精妙配合 ​

Delta 记录的是操作日志而非快照

javascript
// 插入文本"hello"并设置加粗
{ "ops": [{ "insert": "hello", "attributes": { "bold": true } }] }
// 删除3个字符
{ "ops": [{ "delete": 3 }] }
// 保留5个字符（无变更），再插入"world"
{ "ops": [{ "retain": 5 }, { "insert": "world" }] }
  • 极高效的存储，100 次编辑日志远远小于 100 份数据/文档快照
  • 极低计算成本，反向 Delta，而非整个数据快照的重新加载
  • 数据不可变，每一步操作都是增量，天然对协同编辑友好

对比其他编辑器：

  • Quill/Draft：操作日志型（Delta/ContentState），存储高效
  • Slate/TipTap：状态驱动型（树形结构），类型安全
  • TinyMCE/UEditor：快照型（HTML），简单但存储成本高

三、Parchment 文档模型 ​

Blot 的层次结构 ​

架构思想 ​

  • 与 DOM 平行的抽象树
  • Blot 类型系统（类似 DOM Node）
  • 跨浏览器一致性

对比其他编辑器：

  • Quill：Parchment/Blot 抽象层
  • ProseMirror/TipTap：Schema + Node 类型系统
  • Slate：Node + Element 树形结构

关键 Blot 类型 ​

ScrollBlot（文档根） ​

  • 对应 <div class="ql-editor">
  • 管理整个文档的 Delta ↔ DOM 转换

BlockBlot（块级） ​

  • 段落、标题等块级元素
  • 每个块以 \n 结尾

关键 Blot 类型 ​

InlineBlot（内联格式） ​

  • 加粗、斜体、颜色等文本格式
  • 可嵌套

EmbedBlot（嵌入） ​

  • 图片、视频等不可编辑内容
  • 在 Delta 中表示为对象：{ "insert": { "image": "url" } }

Parchment 的优化机制 ​

节点合并

html
<!-- 优化前 -->
<p><strong>Hello</strong><strong>World</strong></p>

<!-- 优化后 -->
<p><strong>HelloWorld</strong></p>

四、实现原理 ​

编辑流程 ​

用户输入 → Delta ​

DOM 变化 → MutationObserver → 计算 Delta → 触发事件

Delta → DOM 渲染 ​

Delta → Blot 操作 → 计算更新范围 → 最小化 DOM 更新 → 优化

MutationObserver 机制 ​

javascript
// 监听 DOM 变化
observer.observe(root, {
  childList: true,
  subtree: true,
  characterData: true
});

// 双向同步
DOM 变化 → 计算 Delta → 更新内部状态

对比其他编辑器：

  • Quill：MutationObserver 监听 DOM → 计算 Delta
  • Slate：React 状态驱动 → 统一 DOM 操作
  • ProseMirror：Transaction 系统 → 状态更新 → 视图渲染

撤销/重做实现 ​

javascript
class History {
  record(change, oldDelta) {
    this.stack.push({
      undo: change.invert(oldDelta), // 反向操作
      redo: change
    });
  }

  undo() {
    this.quill.updateContents(item.undo);
  }
}

基于 Delta 的不可变性，保存操作历史

对比其他编辑器：

  • Quill/Draft：操作反转（invert），存储操作序列
  • Slate：状态快照栈，存储完整状态
  • ProseMirror：Transaction 历史，支持分支历史

五、定制逻辑 ​

自定义 Blot 示例 ​

javascript
class ImageBlot extends Parchment.EmbedBlot {
  static blotName = 'image';
  static tagName = 'img';

  static create(value) {
    const node = super.create(value);
    const wrapper = document.createElement('div');
    wrapper.appendChild(node);
    wrapper.style.display = 'inline-block';
    wrapper.style.maxWidth = '100%';
    wrapper.style.verticalAlign = 'top';
    return wrapper;
  }

  value(domNode) {
    return domNode.src;
  }
}

Parchment.register(ImageBlot);

自定义 Blot 关键步骤 ​

  1. 继承对应的 Blot 基类
  2. 定义 blotName 和 tagName
  3. 实现 create、value
  4. 注册 Blot

六、最佳实践 ​

数据结构设计 ​

✅ 优先使用 Delta

  • 存储、传输都用 Delta 格式
  • 避免直接操作 DOM

✅ 保持不可变性

  • 禁止直接修改 Delta 对象
  • 使用 compose、transform 等方法

七、总结 ​

富文本编辑器的共性设计 ​

1. 数据模型抽象

  • 统一的数据表示（Delta/Node/Schema），与 DOM 解耦
  • 不可变数据结构，支持时间旅行和协同编辑

2. 抽象层设计

  • 文档模型抽象（Parchment/ProseMirror/Slate），屏蔽浏览器差异
  • 类型系统（Blot/Node/Element），保证结构一致性

3. 双向同步机制

  • 用户输入 → 数据模型 → DOM 渲染
  • DOM 变化 → 数据模型 → 状态更新

4. 模块化架构

  • 插件/扩展系统，按需加载功能
  • 命令/操作抽象，统一行为接口

5. 性能优化

  • 最小化 DOM 操作，批量更新
  • 虚拟化渲染，处理大文档
